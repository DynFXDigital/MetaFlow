# Phase 3: Evidence, Burn-down, and Readiness

## Objective

Record authoritative execution evidence and finalize gap closure status and release readiness in `FTR`.

## Tasks

1. Update `doc/ftr/FTR-metaflow-ext.md` with run metadata and per-`TC-*` outcomes.
2. Include durable evidence links for all changed/new high-risk `TC-*` rows.
3. Record fail/skip reasons with issue linkage and owners.
4. Update gap burn-down: closed, open, newly discovered.
5. Document readiness recommendation (`Ready`, `Conditionally Ready`, `Not Ready`) with rationale.

## Exit Criteria

1. Every in-scope high-risk `TC-*` has current run status in `FTR`.
2. Gaps are considered closed only when `FTR` shows passing evidence.
3. Residual risk list is explicit, assigned, and dated.
